## What landed

### Backend (W7) — `src-tauri/src/history/`

- **`store.rs`** — `HistoryStore` opens `%APPDATA%/boothrflow/history.db` with WAL + `synchronous=NORMAL` pragmas. Schema (via `rusqlite_migration`): `utterances` table + FTS5 virtual table over `formatted` + sync triggers. `record()` returns in <1ms after the SQL insert; embedding fires-and-forgets in a background thread so the dictation hot path stays unblocked.
- **`embed.rs`** — OpenAI-compat HTTP client to `/v1/embeddings`. Default `nomic-embed-text` (768 dims) via Ollama. Configurable via `BOOTHRFLOW_EMBED_*` env vars. Vectors packed as little-endian f32 BLOBs in the `embedding` column.
- **`search.rs`** — Cosine similarity + Reciprocal Rank Fusion (k=60) over BM25 (FTS5) + cosine (brute force). Returns `(id, score, source: lexical|semantic|both)` triples. Brute-force semantic search is fine for sub-10k entries; sqlite-vec swap-in is a P4 polish task when histories actually grow.

### IPC contract (W7-D) — `src-tauri/src/commands.rs`

```rust
history_recent(limit: usize) -> Vec<HistoryEntry>
history_search(query: String, limit: usize) -> Vec<SearchResult>
history_paste(id: i64) -> HistoryPasteResult           // pastes wherever focus is
history_delete(id: i64) -> ()
history_clear() -> ()
history_stats() -> HistoryStats                          // total/embedded counts, db_path, embed cfg
quickpaste_paste(id: i64) -> HistoryPasteResult         // palette flow: hide → restore HWND → paste
quickpaste_close() -> ()                                 // dismisses palette without pasting
```

All payload types derive `specta::Type`, so the UI agent can pick up typed bindings via `pnpm gen` once we wire tauri-specta (W3 deferral; not yet done — for now the FE has manual mirror types).

### Quick-paste palette (W8) — `src-tauri/src/quickpaste.rs`

- **Hotkey**: Alt+Meta+H tap. Distinct from the Ctrl+Meta PTT combo. Wired through the same rdev daemon (`hotkey/global.rs`) — added Alt-key state tracking and a rising-edge detection on `KeyH` while Alt+Meta both held.
- **Window**: `index.html#quick-paste`, frameless, transparent, always-on-top, skip-taskbar, 560×360 logical. Pre-warmed at app startup (latency parity with the listen pill).
- **Focus restoration**: process-global atomic stashes `GetForegroundWindow` HWND when the hotkey fires. On paste, the backend hides the palette → calls `SetForegroundWindow(stashed)` → 40ms beat → `ClipboardInjector.inject(text)`. Without this the paste lands inside the palette window.

### Frontend (W8) — minimal

- **`src/lib/state/quickpaste.svelte.ts`** — runes-based reactive store. Holds query, recent, results, selected index. `loadRecent()`, `search(query)`, `pasteSelected()`, `pasteById(id)`, `close()`.
- **`src/lib/quickpaste/QuickPasteApp.svelte`** — input + result list + meta row (style, relative time, embedded-dot). Esc → close, ↑↓ → navigate, Enter / click → paste. Scoped `<style>` block so the UI agent can re-skin without conflict.
- **`src/App.svelte`** — extended hash routing: `#listen-pill` / `#quick-paste` / settings. The `import("$lib/quickpaste/QuickPasteApp.svelte")` is async so the main settings bundle doesn't pull in palette code.

## Senior-eng review

### Architecture — clean

- `HistoryStore` is the single owner of the SQLite connection (in a `parking_lot::Mutex`). Tauri commands access via managed state; session daemon takes an `Option<Arc<...>>`. No lock-held-across-await issues because all DB ops are sync.
- Embedding work is genuinely fire-and-forget: `record()` returns after the insert, the embed thread is named `embed-{id}` for log tracing, and the failure-mode (LLM down / wrong model) is "lexical search keeps working, semantic stays empty until Ollama is available."
- Background thread acquires the DB mutex briefly for the UPDATE — won't starve writers because the inserts are sub-ms.
- `quickpaste.rs` keeps the focus-restore logic platform-isolated under `#[cfg(windows)]`. macOS/Linux versions are no-ops; the pipeline still runs, paste just lands wherever current focus is.

### Things I'd flag in a real PR review

1. **Brute-force semantic scan reads ALL embeddings on every search.** For 100 entries this is 0.3MB and < 1ms. For 10k it's 30MB and ~10ms. For 100k it's 300MB and ~100ms — at which point we need sqlite-vec. **Filed for P4 polish; document the threshold in DECISIONS.**
2. **No retention policy yet.** Database grows forever. Should add a settings field `retention_days` and a daily cleanup pass. **Filed for W7.5 polish.**
3. **`history_paste` (called from main UI, not the palette) doesn't try to restore focus** — it just pastes wherever the user clicked. If they click "paste" in the main window the text lands in the main window. Documented in source. **Filed: should we add a prev-focus stash for the main UI too?**
4. **App-context fields (`app_exe`, `window_title`) are always None.** Plumbing is there; W5 fills it.
5. **FTS5 porter stemmer may over-stem.** "running" → "run" merges with "runs". Acceptable default; offer `tokenize='unicode61'` (no stemming) toggle later if it bites.
6. **Embedding model swap doesn't re-embed old rows.** `embedding_dims` column tracks dim per row; queries filter by NULL embedding only. If user switches `BOOTHRFLOW_EMBED_MODEL` from 768-dim to 1024-dim, semantic search will return empty for old rows (cosine of mismatched dims is zero in our impl). **Filed: re-embed pass + explicit dim check in cosine.**
7. **Palette window doesn't auto-close on `window:blur`** in some cases — Tauri's blur events on Windows can be flaky. Worked around with a `setTimeout(close, 60)` so click-to-paste fires first. May need to revisit if users report ghost palettes.

### What's NOT a concern

- ✅ ADR-009 honored (PLAN.md called for `bge-small-en-v1.5` 384-dim; we adapted to `nomic-embed-text` 768-dim because Ollama was the natural delivery vehicle and reused the OpenAI-compat HTTP pattern from W4).
- ✅ Focus restore correctly clears the stashed HWND on paste (so a stale value can't hijack a future call).
- ✅ ClipboardInjector preserves the user's clipboard contents around the paste, same as the dictation-hot-path injection.

## PM review — manual test plan

Once you pull and run:

1. **Persistence round-trip**:
   - Dictate three things.
   - Open settings UI → confirm they're in "Recent transcripts" (the existing list in App.svelte already calls `dictationStore` which we'll wire to `history_recent` next iteration; for now check the new history exists by stopping `pnpm dev:msvc` and running `sqlite3 "%APPDATA%\boothrflow\history.db" "SELECT id, formatted FROM utterances;"`).
   - Restart `pnpm dev:msvc`. Open palette (Alt+Win+H). Your dictations should be there.

2. **Quick-paste smoke**:
   - Open Notepad. Focus the editing area.
   - Press Alt+Win+H. Palette appears (top-third of monitor).
   - Type a word from a recent dictation. Watch the list filter.
   - Arrow ↓/↑ to navigate. Press Enter.
   - Notepad should receive focus, paste should land at cursor.

3. **Esc cancels**:
   - Open palette, type something, press Esc. Palette closes, no paste.

4. **Embedding flow**:
   - Make sure Ollama has `nomic-embed-text` pulled: `ollama pull nomic-embed-text`.
   - Dictate. Wait ~1 second. Open palette. The entry should show a small green dot (●) meaning embedding is stored.
   - Type a query that's semantically related but lexically different (e.g. you said "let's grab lunch" → search for "food"). The semantic match should surface even without keyword overlap.

5. **No-Ollama graceful degrade**:
   - Stop Ollama (`taskkill /im ollama.exe /f` on Windows).
   - Restart `pnpm dev:msvc`.
   - Dictate. The pipeline still works (raw transcript pastes — LLM cleanup falls back).
   - Open palette. Recent shows entries WITHOUT the green dot (no embeddings).
   - Lexical search still works.

6. **Focus weirdness check**:
   - Open the palette while Notepad is focused. Pick an entry. → Should paste to Notepad.
   - Open the palette while VS Code is focused. Pick an entry. → Should paste to VS Code.
   - If it pastes into the palette itself, the focus-restore is broken — flag with logs.

## What this unlocks for the UI agent

The IPC contract is the deliverable; everything in `commands.rs` is what they consume. Specifically:

```typescript
// Tauri commands available to the UI:
invoke<HistoryEntry[]>("history_recent", { limit?: number })
invoke<SearchResult[]>("history_search", { query: string, limit?: number })
invoke<HistoryPasteResult>("history_paste", { id: number })
invoke<HistoryPasteResult>("quickpaste_paste", { id: number })
invoke("quickpaste_close")
invoke("history_delete", { id: number })
invoke("history_clear")
invoke<HistoryStats>("history_stats")

// Types they can mirror or generate via specta:
type HistoryEntry = { id, captured_at, raw, formatted, style, app_exe, window_title,
                      duration_ms, llm_ms, has_embedding }
type SearchResult = { entry: HistoryEntry, score: number, source: "lexical"|"semantic"|"both" }
type HistoryStats = { total_entries, embedded_entries, db_path, embed_endpoint, embed_model }
```

The minimal `QuickPasteApp.svelte` and `quickpaste.svelte.ts` are reference implementations they can replace with anything that hits the same commands.
