## What's NOT verified (needs your hands)

This stuff can only be verified by an actual human with an actual mic and actual target apps:

1. **Audio quality through your specific mic** — wired headset vs Bluetooth (HFP codec switch is ugly), USB mic, laptop builtin.
2. **Hotkey under games / fullscreen apps** — does Ctrl+Win get eaten by a game's input grab?
3. **Hotkey when AV/EDR scans the LL_KEYBOARD_HOOK** — Defender SmartScreen / corporate AV may flag.
4. **Whisper accuracy on your voice** with tiny.en. Expect ~70-80% accuracy; large-v3-turbo is much better but ~20× slower on CPU.
5. **Clipboard injection across the apps you actually use** — Notepad, Word, Slack, Chrome, VS Code, Discord, Excel, your IDE, password fields, the works.
6. **Pill window positioning** — multi-monitor, fractional DPI, ultrawide.
7. **Latency on your hardware**. tiny.en on a modern CPU should be <500ms for 5s of audio. Anything else and we have work to do.

## Setup (~5 minutes)

```powershell
# 1. Install dev deps (one-time)
winget install Rustlang.Rustup
rustup toolchain install stable
winget install Microsoft.VisualStudio.2022.BuildTools
winget install LLVM.LLVM

# 2. Download Whisper tiny.en (~75MB)
$dest = "$env:APPDATA\boothrflow\models\ggml-tiny.en.bin"
New-Item -ItemType Directory -Force (Split-Path $dest) | Out-Null
Invoke-WebRequest "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-tiny.en.bin" -OutFile $dest

# 3. Build + run
cd "C:\Users\ericb\Coding Projects\boothrflow"
pnpm install
pnpm dev:msvc        # ← this is the cmd that boots tauri dev with the right env
```

First Tauri dev boot compiles whisper.cpp from C++ source (~5-10 min). Subsequent runs are <30s. The main settings window opens; the listen pill exists invisibly until you press the hotkey.

## First-run smoke test (5 minutes)

1. Boot the app: `pnpm dev:msvc`. Settings window opens. Status: "idle".
2. **Notepad test.** Open Notepad. Focus the text area. Hold `Ctrl + Win`, say "**hello world this is a test of voice dictation**", release.
   - **Expected:** the listen pill appears bottom-center of your monitor while held. On release, the pill hides; ~500ms later the transcribed text appears in Notepad. The settings window also shows the transcript in the "Live transcript" panel.
   - **Failure modes to file:** pill doesn't appear; pill in wrong position; nothing pastes; partial paste; clipboard contents lost; long delay (>3s).
3. **Tray menu.** Right-click the tray icon. Click "Pause". Hold `Ctrl+Win` again — nothing should happen. Click "Resume". Hotkey works again.
4. **Status text.** Watch the tray tooltip (hover the tray icon). It should say "boothrflow — listening" while held, "boothrflow — idle" otherwise.

If those four pass, the spine works. Move to the broader matrix below.

## Manual test matrix

Run each of these and report what worked / what didn't.

### Apps to dictate into

- [ ] **Notepad** — sanity baseline
- [ ] **Slack** desktop (Electron) — should work via clipboard paste
- [ ] **Discord** — Electron, similar
- [ ] **Chrome / Edge** — address bar, search box, web text fields, gmail compose
- [ ] **VS Code / Cursor** — editor body, terminal, find box, settings search
- [ ] **Word** — body, headers, comments
- [ ] **Excel** — cell editing (it strips clipboard formats, watch for weirdness)
- [ ] **Outlook** — compose body
- [ ] **Your terminal** (Windows Terminal / pwsh / cmd / WSL)

### Edge cases

- [ ] **Password field** in Chrome: dictate "secret123" into a password input. Most browsers block clipboard paste in password fields — the paste should fail and we should emit a clear error to the main window. (W3+ would add the typing fallback for this case.)
- [ ] **Elevated app**: open Task Manager (run as admin). Try to dictate into its filter box. Without us also being elevated, SendInput will be blocked silently (UIPI). Report what happens.
- [ ] **Game / fullscreen exclusive**: alt-tab to a game running fullscreen-exclusive. Hotkey may not register. (Acceptable failure for v0.)
- [ ] **Multi-monitor**: the pill should appear on the monitor your focused window is on. If you have a 2nd monitor, drag a window over there, focus it, dictate.
- [ ] **DPI scaling**: 4K display at 150% scale, ultrawide, etc.
- [ ] **Bluetooth headset**: speak into one. Audio mode might switch to HFP (8/16kHz) which will sound terrible to Whisper. Document if accuracy tanks.

### Robustness

- [ ] **Hold hotkey for 30 seconds**: see if the daemon survives, the pill stays open, Whisper handles 30s of audio.
- [ ] **Tap-and-release fast** (<500ms): minimum-duration audio. We short-circuit <0.5s; you should see "skipping (short audio)" in the logs.
- [ ] **Hotkey while another app is fullscreen**: Edge browser fullscreen, VS Code fullscreen.
- [ ] **Mic permissions revoked mid-session**: rare but should not crash.
- [ ] **Clipboard preservation**: copy something to clipboard before dictating. After dictation, your clipboard should still hold the original (with the paste happening to the focused app). Acceptable v0 limitation: only text is preserved; images/files on clipboard get clobbered.
- [ ] **Cancel via Esc**: NOT YET WIRED. Holding hotkey + pressing Esc has no effect in v0. Document if you need this — it's a quick W4 add.

`pnpm dev:msvc` prints structured tracing to the terminal. You should see:

```
INFO boothrflow_lib: audio: device=Headset (Sennheiser GSP 600) rate=48000 channels=2 format=F32
INFO boothrflow_lib: hotkey: starting rdev listener (Ctrl+Meta hold-to-talk)
INFO boothrflow_lib: whisper: model loaded
INFO boothrflow_lib: session daemon ready — Ctrl+Meta to dictate

(press hotkey, speak, release)

INFO boothrflow_lib: captured: 142 frames, 4.55s, peak -8.3 dBFS
INFO boothrflow_lib: transcript (380 ms): "hello world this is a test of voice dictation"
```

If you see `consumer slow, dropping a 30ms frame` repeating: audio path is backed up; expected on lower-end hardware while Whisper warms up.

If you see `audio capture thread errored`: WASAPI failed; restart and report.

If you see `rdev::listen errored`: AV/EDR blocked the hook; we'll need to sign + submit to MS reputation.

## Known limitations / non-blockers

1. **No LLM cleanup pass yet** — transcripts paste raw, including any "uh"s and disfluencies. Lands in P2 W4.
2. **No styles / per-app rules** — same raw output regardless of target. P2 W5.
3. **No Esc to cancel mid-utterance** — release the hotkey to abort. P2 polish.
4. **No history persistence** — transcripts visible in the main window until next launch. P3.
5. **No settings persistence** — pause state, model path, etc. reset on restart. P3.
6. **Pill appears at hardcoded bottom-center**, not near caret. Caret position via UIA is P3.
7. **Clipboard restore only preserves text formats**. Images/files get clobbered. Documented in source.
8. **Tray icon is the default app icon**, not state-color-coded. v1 polish.
9. **`SerializedWhisperSttEngine` struct is unused** — leftover from an earlier design pass. Will delete next PR.
